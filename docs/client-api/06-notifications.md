# Notifications

Notification endpoints allow clients to retrieve and manage the authenticated user's notification stream. Notifications are generated by social interactions (mentions, follows, favourites) and by Corundum-specific protocol events.

## Endpoints

### GET /api/v1/notifications

Returns the authenticated user's notifications, newest first.

**Authentication:** Required (`read:notifications` scope).

**Query Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `max_id` | string | — | Return results older than this notification ID |
| `min_id` | string | — | Return results newer than this notification ID |
| `limit` | integer | 15 | Maximum number of results (max 30) |
| `exclude_types[]` | array of strings | — | Exclude notifications of these types (see type list below) |
| `types[]` | array of strings | — | Include only notifications of these types. Mutually exclusive with `exclude_types[]`. |
| `account_id` | string (IPNS name) | — | Return only notifications from this account |

`types[]` and `exclude_types[]` are mutually exclusive. If both are provided, the operator returns a 422 error.

**Returns:** Array of Notification entities, newest first.

**Errors:**

| Code | Error | Description |
|------|-------|-------------|
| 401 | `unauthorized` | No valid token provided |
| 403 | `insufficient_scope` | Token lacks `read:notifications` scope |
| 422 | `unprocessable_entity` | Both `types[]` and `exclude_types[]` were specified |

---

### GET /api/v1/notifications/:id

Returns a single notification by ID.

**Authentication:** Required (`read:notifications` scope).

**Path Parameters:**

| Parameter | Description |
|-----------|-------------|
| `:id` | The notification ID |

**Returns:** Notification entity.

**Errors:**

| Code | Error | Description |
|------|-------|-------------|
| 401 | `unauthorized` | No valid token provided |
| 403 | `insufficient_scope` | Token lacks `read:notifications` scope |
| 404 | `not_found` | No notification with that ID, or it belongs to a different user |

---

### POST /api/v1/notifications/clear

Deletes all notifications for the authenticated user. This action is irreversible.

**Authentication:** Required (`write:notifications` scope).

**Request Body:** None.

**Returns:** Empty object `{}`.

**Errors:**

| Code | Error | Description |
|------|-------|-------------|
| 401 | `unauthorized` | No valid token provided |
| 403 | `insufficient_scope` | Token lacks `write:notifications` scope |

---

### POST /api/v1/notifications/:id/dismiss

Dismisses (deletes) a single notification by ID.

**Authentication:** Required (`write:notifications` scope).

**Path Parameters:**

| Parameter | Description |
|-----------|-------------|
| `:id` | The notification ID to dismiss |

**Request Body:** None.

**Returns:** Empty object `{}`.

**Errors:**

| Code | Error | Description |
|------|-------|-------------|
| 401 | `unauthorized` | No valid token provided |
| 403 | `insufficient_scope` | Token lacks `write:notifications` scope |
| 404 | `not_found` | No notification with that ID, or it belongs to a different user |

## Notification Types

Corundum supports the standard Mastodon notification types plus additional types for Corundum-specific protocol events.

| Type | Trigger |
|------|---------|
| `mention` | Someone mentioned you in a status |
| `reblog` | Someone reblogged (boosted) one of your statuses |
| `favourite` | Someone favourited one of your statuses |
| `follow` | Someone followed you |
| `follow_request` | Someone sent a follow request (only for locked accounts) |
| `poll` | A poll you voted in, or a poll you created, has ended |
| `status` | Someone you enabled notifications for posted a new status |
| `corundum.block_enforcement` | A reply you submitted was rejected because you are blocked by the post's author |
| `corundum.migration_incoming` | An account has migrated to or from your operator |
| `corundum.curation_flagged` | A curation feed has applied a label to one of your statuses |

### corundum.block_enforcement

This notification type is unique to Corundum. When a user attempts to reply to a post authored by someone who has blocked them, the inter-operator protocol rejects the submission with `BLOCKED_USER`. Rather than silently discarding the rejected reply, Corundum delivers a `corundum.block_enforcement` notification to the sender.

This is an intentional transparency design: the blocked user receives explicit feedback that their reply was not delivered, rather than believing it was posted when it was not. The notification does not reveal who the block originates from; it indicates only that the submission was rejected due to a block relationship.

### corundum.migration_incoming

Delivered when an account completes a migration that involves the local operator — either an account migrating in (a new account that has chosen this operator) or an account migrating out (an existing local account that has designated a new operator). The notification includes a reference to the migrating account.

### corundum.curation_flagged

Delivered when a curation feed that the user subscribes to (or that is applied by the operator) applies a label to one of the user's own statuses. The notification includes the label type and the curation feed that applied it. Users can use this to initiate an appeal via the curation endpoints.

## Notification Entity

Notification entities include the following fields:

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Notification ID |
| `type` | string | Notification type (see table above) |
| `created_at` | string (ISO 8601) | When the notification was created |
| `account` | Account entity | The account that triggered the notification |
| `status` | Status entity or null | The relevant status, if applicable to this notification type |
| `corundum` | object or null | Corundum-specific fields; present only for `corundum.*` notification types |

For `corundum.block_enforcement`, the `corundum` object contains:

```json
{
  "corundum": {
    "rejected_status_cid": "bafyreib4gy7..."
  }
}
```

For `corundum.curation_flagged`, the `corundum` object contains:

```json
{
  "corundum": {
    "label": "content-warning",
    "curation_feed_id": "bafyreicde8..."
  }
}
```
